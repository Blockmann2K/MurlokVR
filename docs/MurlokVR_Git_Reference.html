Branch Naming GitHub Flow

Prefixes
feat/New Feature
fix/Bug Fix
docs/Documentation Only
refactor/Restructure, No New Logic
test/Tests Only
chore/Maintenance, Config, ...
experiment/Uncertain — May Be Discarded
MurlokVR Examples
→feat/simple-hmd
→feat/runtime-core
→feat/shared-memory-ipc
→feat/openvr-adapter
→fix/sensor-drift
→experiment/slam-prototype
Rules
—All Lowercase!
—Hyphens Only, No Underscores or Spaces!
—Short but Descriptive — Readable at a Glance!
—Branch Off main & Merge Back When Solid Yay!

Optional Body — Explain WHY, Not What!

Rules
—Type and Scope Lowercase! — "feat" Not "Feat"
—Imperative Tense — "Add" Not "Added"
—Scope Is Optional but Useful for a Big Project Like MurlokVR
—Keep the Description Short — Under ~72 Characters
—Body Explains WHY, Not What — the Diff Already Shows the What
